# 이분 탐색

## 개요

정렬된 배열에서 원소 위치를 O(log N)에 찾는 기법. 더 중요한 응용은 **Parametric Search**("결정 문제를 답에 대해 이분").

## `bisect` 사용법

```python
from bisect import bisect_left, bisect_right, insort

arr = [1, 2, 2, 2, 4, 5]
bisect_left(arr, 2)   # 1  (같은 값 왼쪽)
bisect_right(arr, 2)  # 4  (같은 값 오른쪽)
insort(arr, 3)        # 정렬 유지하며 삽입

# 카운트: 특정 값의 개수
def count(arr, x):
    return bisect_right(arr, x) - bisect_left(arr, x)
```

## 직접 구현 — lower_bound

```python
def lower_bound(arr, target):
    """arr[i] >= target인 최소 i"""
    lo, hi = 0, len(arr)
    while lo < hi:
        mid = (lo + hi) // 2
        if arr[mid] < target:
            lo = mid + 1
        else:
            hi = mid
    return lo
```

`hi = len(arr)` (exclusive)로 두는 게 가장 덜 헷갈림.

## Parametric Search

"이 값 K로 가능한가?"가 **단조(monotone)**하다면, K에 대해 이분.

전형: **최소/최대 가능한 K를 찾아라**.

```python
def parametric(lo, hi, check):
    """
    check(x) == True이면 OK. 가장 작은 OK를 찾는 패턴.
    """
    while lo < hi:
        mid = (lo + hi) // 2
        if check(mid):
            hi = mid
        else:
            lo = mid + 1
    return lo
```

### 예: 나무 자르기 (백준 2805 유형)

높이 H로 절단 시 확보 목재 ≥ M이 되는 최대 H.

```python
def max_height(trees, M):
    lo, hi = 0, max(trees)
    # "H 이하 자를 때 얻는 양 ≥ M"이 단조 감소 (H 커질수록 감소)
    # 가장 큰 H를 찾는 문제 → 전형 upper 이분.
    while lo < hi:
        mid = (lo + hi + 1) // 2        # +1: upper용
        if sum(max(0, t - mid) for t in trees) >= M:
            lo = mid
        else:
            hi = mid - 1
    return lo
```

"하나 크게 잡아라"(upper) 버전은 `+1`과 `lo = mid`, `hi = mid - 1`의 조합. 경계 실수 가장 많은 지점.

## 실수(float) 이분

종료 조건을 "차이 < ε"로. 50~100회 반복이면 충분.

```python
def bisect_float(lo, hi, check, eps=1e-9):
    for _ in range(100):
        mid = (lo + hi) / 2
        if check(mid):
            hi = mid
        else:
            lo = mid
    return lo
```

고정 횟수가 while + eps보다 안전 (단조성이 느슨해도 문제 안 생김).

## 응용: 답의 범위 이분

- **최솟값 최대화 / 최댓값 최소화** 류 (예: "K개 뿌릴 때 최소 간격 최대")
- **N개를 M개 그룹으로 나눌 때, 각 그룹 최대 합 최소**

공통 구조: "답이 K일 때 조건 만족?"을 **선형/그리디로** 판정 가능하면 전체 O(N log(answer)).

## 함정

- `(lo + hi) // 2` vs `(lo + hi + 1) // 2` (upper/lower 버전).
- `lo = mid` vs `lo = mid + 1` 선택.
- 고정된 방식 하나만 외워 쓰기. 위의 lower_bound 스타일을 기본으로 하면 경계 실수 거의 없음.

## 관련

[[LIS]] O(N log N) 풀이, [[Two Pointers]]와 자주 결합. 답이 정수일 때 더 자주 쓰이고, 기하·물리 문제는 float 이분.

## 실행 예시

```python
from bisect import bisect_left, bisect_right

arr = [1, 2, 2, 2, 4, 5]
print("bisect_left(arr, 2):", bisect_left(arr, 2))     # 1
print("bisect_right(arr, 2):", bisect_right(arr, 2))   # 4
print("2의 개수:", bisect_right(arr, 2) - bisect_left(arr, 2))  # 3

# --- lower_bound 수동 구현 ---
def lower_bound(arr, target):
    lo, hi = 0, len(arr)
    while lo < hi:
        mid = (lo + hi) // 2
        if arr[mid] < target:
            lo = mid + 1
        else:
            hi = mid
    return lo

print("lower_bound 3:", lower_bound(arr, 3))   # 4

# --- 나무 자르기 (Parametric Search) ---
def max_height(trees, M):
    lo, hi = 0, max(trees)
    while lo < hi:
        mid = (lo + hi + 1) // 2
        if sum(max(0, t - mid) for t in trees) >= M:
            lo = mid
        else:
            hi = mid - 1
    return lo

trees = [20, 15, 10, 17]
print("max 절단 높이 (M=7):", max_height(trees, 7))   # 15

# --- 실수 이분 ---
def bisect_float(lo, hi, check, iters=100):
    for _ in range(iters):
        mid = (lo + hi) / 2
        if check(mid):
            hi = mid
        else:
            lo = mid
    return lo

# sqrt(2) 계산
sqrt2 = bisect_float(0, 2, lambda x: x * x >= 2)
print(f"sqrt(2) ≈ {sqrt2:.10f}")   # 1.4142135624
```
