# 좌표 압축

## 개요

값 자체는 크지만(10⁹) **등장하는 값의 종류가 적을 때**(10⁵), 등장 값들을 `0, 1, 2, ...`로 매핑. 배열 기반 자료구조([[Segment Tree]], [[Fenwick Tree]])를 사용 가능하게 해줌.

## 기본 구현

```python
def compress(values):
    sorted_unique = sorted(set(values))
    idx = {v: i for i, v in enumerate(sorted_unique)}
    return [idx[v] for v in values], sorted_unique

# 예: [3, 10, 10, 2, 50] → [1, 2, 2, 0, 3]
```

`bisect_left`를 쓰면 dict 오버헤드 없이:

```python
from bisect import bisect_left

def compress_bisect(values):
    sorted_unique = sorted(set(values))
    return [bisect_left(sorted_unique, v) for v in values], sorted_unique
```

## 응용 1: 구간 쿼리에 좌표 압축

"값이 10⁹까지 올라가지만 쿼리에 등장하는 값은 10⁵개" → 등장 값만 압축 후 [[Segment Tree]].

```python
def range_queries(points, queries):
    # points: 업데이트할 좌표 리스트
    # queries: (l, r) 쌍 리스트, l·r도 큰 값
    all_coords = set(points)
    for l, r in queries:
        all_coords.add(l); all_coords.add(r)
    coords = sorted(all_coords)
    idx = {v: i for i, v in enumerate(coords)}
    # 이후 idx[v]를 0 ~ len(coords)-1 범위의 인덱스로 사용
    ...
```

## 응용 2: 역원 카운트

큰 값 배열의 역원 수를 [[Fenwick Tree]]로 셀 때, 값을 0~N-1로 압축.

```python
from bisect import bisect_left

def inversions(arr):
    compressed = sorted(set(arr))
    rank = {v: i for i, v in enumerate(compressed)}
    n = len(compressed)
    bit = [0] * (n + 1)

    def update(i):
        i += 1
        while i <= n:
            bit[i] += 1
            i += i & -i

    def prefix(i):
        i += 1
        s = 0
        while i > 0:
            s += bit[i]
            i -= i & -i
        return s

    inv = 0
    for x in reversed(arr):
        r = rank[x]
        if r > 0:
            inv += prefix(r - 1)
        update(r)
    return inv
```

## 응용 3: 사각형 면적 합집합 (Sweep Line)

X좌표를 압축한 후 Y 스위핑. 업데이트/쿼리할 값이 유한해지므로 [[Segment Tree]]로 해결.

## 응용 4: 스캔라인 + 압축

"겹치는 구간 최댓값" 유형은 각 좌표를 압축한 후 +1/-1 이벤트 스캔. 좌표 10⁹이어도 이벤트는 2N개뿐.

```python
def max_overlap(intervals):
    events = []
    for l, r in intervals:
        events.append((l, 1))
        events.append((r, -1))   # 반개구간이면 r 그대로, 폐구간이면 r+1
    events.sort()
    cur = best = 0
    for _, d in events:
        cur += d
        best = max(best, cur)
    return best
```

이 경우 별도 압축 자료구조 없이도 좌표 그대로 정렬해서 씀.

## 함정

- **쿼리에 등장하는 값도 함께** 압축해야 함. 업데이트만 압축하고 쿼리는 원본이면 매핑 실패.
- 닫힌 구간 vs 반개구간 경계: `r+1` 처리 여부로 결과가 1씩 어긋남.

## 관련

[[Segment Tree]], [[Fenwick Tree]]와 짝으로 쓰는 경우가 대부분. 전처리 단계의 중요 기법.

## 실행 예시

```python
from bisect import bisect_left

# --- 기본 압축 ---
def compress(values):
    sorted_unique = sorted(set(values))
    idx = {v: i for i, v in enumerate(sorted_unique)}
    return [idx[v] for v in values], sorted_unique

compressed, coords = compress([3, 10, 10, 2, 50, 2])
print("압축 결과:", compressed)    # [1, 2, 2, 0, 3, 0]
print("좌표 매핑:", coords)         # [2, 3, 10, 50]

# --- 역원 카운트 (Fenwick + 좌표 압축) ---
def inversions(arr):
    compressed = sorted(set(arr))
    rank = {v: i for i, v in enumerate(compressed)}
    n = len(compressed)
    bit = [0] * (n + 1)
    def update(i):
        i += 1
        while i <= n:
            bit[i] += 1
            i += i & -i
    def prefix(i):
        i += 1
        s = 0
        while i > 0:
            s += bit[i]
            i -= i & -i
        return s
    inv = 0
    for x in reversed(arr):
        r = rank[x]
        if r > 0:
            inv += prefix(r - 1)
        update(r)
    return inv

print("역원 ([10^9, 10^6, 5, 999]):", inversions([10**9, 10**6, 5, 999]))   # 6 (완전 역순)

# --- 최대 겹치는 구간 수 ---
def max_overlap(intervals):
    events = []
    for l, r in intervals:
        events.append((l, 1))
        events.append((r, -1))
    events.sort()
    cur = best = 0
    for _, d in events:
        cur += d
        best = max(best, cur)
    return best

print("최대 겹침:", max_overlap([(1, 5), (2, 6), (3, 7), (8, 10)]))   # 3
```
