 
**프로세스 간 통신**을 의미하며, 서로 다른 프로세스가 데이터를 주고받거나 동기화하는 메커니즘이다.

---

## 주요 IPC 방식

### 1. 파이프 (Pipe)

- 단방향 통신 (부모-자식 프로세스 간)
- 익명 파이프는 관련 프로세스끼리만 사용 가능
- 간단하지만 유연성이 낮음

### 2. Named Pipe (FIFO)

- 파일 시스템에 이름이 있는 파이프
- 관련 없는 프로세스 간에도 통신 가능
- 단방향 또는 양방향 설정 가능

### 3. 메시지 큐 (Message Queue)

- 커널이 관리하는 큐에 메시지를 넣고 꺼내는 방식
- 비동기 통신에 적합
- 메시지 단위로 데이터 구분이 명확함

### 4. 공유 메모리 (Shared Memory)

- **가장 빠른 IPC 방식**
- 여러 프로세스가 동일한 메모리 영역에 접근
- 동기화(세마포어, 뮤텍스)가 필수
- 대용량 데이터 공유에 적합

### 5. 소켓 (Socket)

- 네트워크 통신 가능 (로컬/원격 모두)
- TCP/UDP 또는 Unix Domain Socket 사용
- 가장 범용적인 방식

### 6. 시그널 (Signal)

- 프로세스에 이벤트 알림
- 제한된 정보만 전달 가능 (인터럽트 성격)
- SIGKILL, SIGTERM 등

### 7. ZeroMQ (ØMQ)

- 고성능 비동기 메시징 라이브러리
- 소켓 API를 추상화하여 다양한 통신 패턴 제공
- 브로커 없이 P2P 통신 가능 (brokerless)
- **지원 패턴**:
    - REQ/REP: 요청-응답
    - PUB/SUB: 발행-구독
    - PUSH/PULL: 파이프라인 (작업 분배)
    - PAIR: 1:1 양방향
- **전송 방식**:
    - `tcp://` : 네트워크 통신
    - `ipc://` : Unix Domain Socket (로컬 프로세스 간)
    - `inproc://` : 스레드 간 통신
- 내부 PC 간 통신, 마이크로서비스, 분산 시스템에 적합
- Python: `pyzmq` 라이브러리 사용

## IPC 방식 비교

|방식|속도|복잡도|네트워크 지원|용도|
|---|---|---|---|---|
|파이프|빠름|낮음|X|부모-자식 프로세스|
|Named Pipe|빠름|낮음|X|로컬 프로세스 간|
|메시지 큐|중간|중간|X|비동기 메시지 전달|
|공유 메모리|최고|높음|X|대용량 데이터 공유|
|소켓|중간|중간|O|범용 통신|
|시그널|빠름|낮음|X|이벤트 알림|
|ZeroMQ|빠름|중간|O|분산 시스템, 내부 통신|

## 백엔드 관점에서의 활용

### 실무에서 자주 사용하는 방식

|상황|추천 방식|
|---|---|
|FastAPI 워커 간 통신|Redis Pub/Sub, 메시지 큐|
|마이크로서비스 간 통신|HTTP/gRPC, 메시지 브로커|
|같은 서버 내 프로세스 통신|Unix Socket, 공유 메모리|
|내부 PC/서버 간 통신|ZeroMQ (ipc://, tcp://)|
|분산 시스템 이벤트 처리|Kafka, RabbitMQ|

### 참고사항

- 실무에서는 직접적인 OS-level IPC보다 **Redis, RabbitMQ, Kafka** 같은 메시지 브로커를 더 많이 사용함
- Python에서는 `multiprocessing` 모듈로 파이프, 큐, 공유 메모리 등을 쉽게 사용 가능
- FastAPI + Uvicorn 환경에서는 워커 간 상태 공유 시 Redis 등 외부 저장소 권장
