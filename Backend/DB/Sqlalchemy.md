## SQLAlchemy Engine 설정

- **`pool_pre_ping=True`**: 커넥션 풀에서 연결을 꺼낼 때마다 `SELECT 1`로 연결 상태를 먼저 확인. 끊어진 연결이면 자동으로 폐기하고 새 연결 생성. 약간의 오버헤드가 있지만 Swarm 환경에선 필수.
- **`pool_recycle=180`**: 연결 생성 후 180초(3분)가 지나면 강제로 폐기하고 새 연결로 교체. Swarm 내부 네트워크가 유휴 연결을 몰래 끊는 걸 선제적으로 방지. 일반 환경은 3600(1시간)도 괜찮지만 Swarm에선 짧게 잡는 게 안정적.
- **`pool_size=10`**: 풀에 상시 유지하는 연결 수. 동시 요청이 이 수를 넘으면 `max_overflow`까지 추가 생성. 트래픽에 따라 조절.
- **`max_overflow=20`**: `pool_size` 초과 시 추가로 생성 가능한 연결 수. 최대 동시 연결은 `pool_size + max_overflow = 30`개. 사용 후 자동 폐기.
- **`pool_timeout=30`**: 풀에서 연결을 가져올 때 최대 대기 시간(초). 30초 안에 연결을 못 얻으면 `TimeoutError` 발생.
