
기존에 라벨링된 이미지 호출로직
1. 프론트에서 라벨링 저장(이미지 + 라벨링 이후 썸네일 크기로 스케일링)
2. 백엔드에서 라벨링 이미지 존재 여부 체크
3. 라벨링 이미지 존재시 프론트에서 labeled/path에서 라벨링된 이미지 조회

단점
1. 기존 썸네일도 저장하고 이후 라벨링된 이미지도 저장
	- 저장용량 2배
2. 라벨링된 이미지의 경우 캐싱된 이미지를 가져오면 최신라벨링 반영이 안됨
	- 캐싱활용 불가

개선
- 썸네일과 라벨을 분리
- 썸네일 이미지는 항상 캐싱된 이미지를 가져옴 -> CDN 캐싱
- 라벨링만 그리면 저장용량 기존 라벨 이미지의 1/10 크기도 안됨
